### 👤 About

I build ML applications, understand the hardware they run on, and create tools that help AI systems reason better. My work spans three layers:

- **Applied ML** — predictive models with Scikit-learn, XGBoost, and interactive dashboards
- **Systems** — C++ CPU simulation with hand-tuned AVX2 kernels (I care about what happens under `model.predict()`)
- **Agents** — AI coding agent configuration, custom extensions, multi-provider LLM orchestration

### 📌 Projects

⚙️ **[cpu-framework](https://github.com/raindragon14/cpu-framework)**
C++ CPU simulation framework — hand-tuned AVX2 GEMM kernels, xoshiro256+ PRNG, Zen 2 microarchitecture optimization. This is where I explore what a CPU actually does when running ML workloads.

📈 **[Laptop_Predict](https://github.com/raindragon14/Laptop_Predict)**
ML regression app that predicts laptop prices from hardware specs. Compares Scikit-learn, XGBoost, and LightGBM. Deployed via Streamlit. A practical example of the full ML workflow: data → model → UI.

📊 **[JatimDatathon](https://github.com/raindragon14/JatimDatathon)**
Data analysis and visualization project from Jatim Datathon. ETL, exploratory analysis with Pandas and Seaborn, interactive dashboard with Plotly and Streamlit.

🔧 **[pi-agent-dotfiles](https://github.com/raindragon14/pi-agent-dotfiles)**
Portable AI coding agent configuration with one-line install. Custom TypeScript extensions (web search, docs lookup, structured reasoning, entropy monitoring), prompt templates, and multi-provider LLM definitions. Everything I know about making AI agents useful, version-controlled.
